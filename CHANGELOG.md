# CHANGELOG

twsnmpfc のリリース変更履歴です。

---

## [v1.71.1] - 2026-07-13

### 🐛 バグ修正

- **メール送信**: SMTP認証が機能していなかった不具合を修正
- **メール送信**: 指定したポート番号が go-mail のデフォルトポートに上書きされる不具合を修正

---

## [v1.71.0] - 2026-07-01

### 🔒 セキュリティ修正

- **CVE-2025-22869** : Fluent Bit プラグインの `golang.org/x/crypto` を v0.45.0 にアップグレード
- **CVE-2026-40611** : `github.com/go-acme/lego/v4` を v4.34.0 にアップグレード
- **CVE-2026-29181** : `go.opentelemetry.io/otel` を v1.41.0 にアップグレード
- DOMPurify のアップグレード（spa / pwa）
- js-yaml、ws のアップグレード（Dependabot アラート対応）
- Lego v2 / square/go-jose.v2 を削除し Lego v4 に統一
- axios / tmp の依存関係を更新し、axios を 0.27.2 に固定
- アイコンパスのサニタイズ、OAuth2 ステートの保護、ログへの機密フィールド出力抑制

### 🛡️ コード品質・脆弱性対応（CodeQL）

- SSRF 警告の修正（ポーリングパッケージ、`ValidateURL` エクスポート）
- ネットワークリクエスト非制御データ（SSRF）警告の修正
- メールコンテンツインジェクション警告の修正

### 🔧 内部改善

- `statik` の利用を Go 標準の `embed` に変更（サードパーティ依存を削減）
- Fluent Bit プラグイン（gopsutil 入力・twsnmp 入出力）を廃止・削除
- マップオブジェクトのサイズをスケールダウンするマイグレーション機能（`migrateMapSize`）を実装

### 🐛 バグ修正

- `package-lock.json` から不要な `buffer` モジュールを削除

---

## [v1.70.0] - 2026-06-09

MQTTレポート対応

- MQTT トピックおよびクライアントレポート機能を実装
- MQTT トピックのコピーおよびポーリング追加通知を v-snackbar に統一
- MQTT ポーリング追加時の送信元 IP アドレスからの対象ノード自動選択
- MQTT stats の自動クリーンアップ（ログ保持日数に基づく）
- ノードおよびネットワーク単位でカスタム SNMP ポート番号を設定可能に
- ノードロック機能の追加（マップ上での誤移動防止）
- アイコンへのツールチップ追加（アクセシビリティ向上）
